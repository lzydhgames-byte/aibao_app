# 爱宝（Aibao）AI 儿童故事 App — 产品设计文档（MVP）

- 文档版本：v1.0
- 创建日期：2026-04-28
- 范围：一期 MVP；二期及以后内容仅在末尾"未来路线图"中概述
- 受众：产品 / 设计 / 工程 / 内容安全团队

---

## 1. 产品定位与核心价值

### 1.1 一句话定位
> 爱宝是一只会百变的熊猫小机器人，住在每个孩子的手机里，用 AI 为孩子生成专属故事，记得孩子的喜好和经历，让每次听故事都像和老朋友一起冒险。

### 1.2 核心差异化（卖点）
1. **🐼 百变伙伴 IP**：本体是熊猫小机器人爱宝，在每个故事里以适配形态出现（奥特曼搭档、小恐龙、太空员…），形象恒定但玩法万变。
2. **👶 孩子是主角**：故事永远以孩子为 C 位，爱宝是伙伴。
3. **🧠 有记忆的 AI**：爱宝记得孩子的喜好、害怕的事、家人名字、听过的故事，让孩子和家长都觉得"它认识我们"。
4. **📺 追剧感**：默认故事间有彩蛋串联；家长可主动开启"连续剧模式"。
5. **🎙️ 沉浸式音频**：TTS 朗读 + BGM + 音效，专为睡前场景优化。

### 1.3 不做什么（YAGNI 边界）
- 不做插图/视觉化故事（一期）
- 不做孩子直接语音交互（一期）
- 不做多孩子档案切换（一期只支持 1 个孩子）
- 不做真·后台主动推送（一期用"打开时呈现"伪推送替代）

## 2. 用户与场景

### 2.1 双层用户结构
- **操作者：家长** —— 注册、设置档案、输入一句话需求、**确认 AI 提案的大纲卡**（Plan 11A 起，替代手填参数）、付费。
- **聆听者：孩子** —— 通过爱宝听故事，是体验的真正主体。一期不与 App 交互。

### 2.2 核心场景
- **睡前哄睡**（主场景，60%+）—— 3/5/8 分钟温馨故事配 BGM。
- **路上/通勤**（次场景）—— 车上、走路时让孩子有事做。
- **白天玩耍/认知**（次场景）—— 科普、探险类风格。

## 3. 九文件架构（产品大脑）

爱宝的 AI 系统采用 OpenClaude 风格的九文件人格架构。每个文件承担明确职责，跨会话持久化，让 AI 拥有一致人格和长期记忆。

### 3.1 SOUL.md —— 价值观、行为准则、决策原则
- **温柔陪伴**：永远不吓到孩子，不留下任何可能引发噩梦的画面。
- **孩子是主角**：每个故事中孩子必须是被赋能的、做出关键决定的角色，爱宝是伙伴而非主角。
- **家长是合作伙伴**：尊重家长的教育意图，不替代家长的判断。
- **真实与温度**：故事里的情感（害怕、想家、不舍）真实但有出口，传递"会好的"。
- **记忆即关爱**：每一次"我记得"都是爱宝表达爱的方式。

### 3.2 IDENTITY.md —— 名字、角色、说话风格
- **名字**：爱宝
- **形象**：酷似大熊猫的可爱 AI 小机器人，圆耳朵、黑眼圈、胸口有一块发光的能量片
- **角色定位**：百变伙伴（C 方案）—— 本体恒定，在故事里可变形为适配各类故事场景的形态。
- **说话风格**：软糯、亲切，爱用拟声词（"咻——""哇哦！"），常用昵称称呼孩子，会"邀请"孩子做决定（"小宇你觉得我们该往左还是往右走呀？"）。
- **永远的姿态**：仰望孩子、为孩子鼓掌、跟随孩子的决定。

### 3.3 AGENTS.md —— 工作规则、安全边界、记忆管理
- **内容安全红线**（详见第 7 章）
- **记忆写入规则**：什么样的信息进入 MEMORY，什么是临时上下文。
- **何时该问家长**：当孩子档案中的关键信息缺失、当生成的故事可能触碰边界时。
- **子 agent 分工**：
  - 需求理解 agent（解析家长一句话输入）
  - 故事生成 agent（产出文本）
  - 串联编剧 agent（注入彩蛋/接续连续剧）
  - 安全审核 agent（生成后过滤）
  - TTS+BGM 编排 agent（合成音频）

### 3.4 USER.md —— 用户画像与上下文
**双层结构：**

**家长层：**
- 账号信息、订阅状态
- 偏好（喜欢的故事风格、教育主题倾向、是否开启连续剧模式）
- 反馈历史（点赞/不喜欢的故事）

**孩子层：**
- **首次必填**：昵称、性别、年龄/生日
- **BOOTSTRAP 仪式中由爱宝采访收集**：性格关键词、喜欢的角色/IP、害怕的东西、家庭成员名字
- **后续逐步沉淀**：兴趣爱好、教育主题命中率、当前面对的成长挑战
- **当前情境**（动态）：当下时段（睡前/白天）、本次会话的需求

### 3.5 TOOLS.md —— 工具策略与成本控制
- **LLM**：故事生成主模型（具体选型见技术架构阶段）
- **TTS**：单声音方案（一期），预留多声音架构
- **BGM/音效库**：自有素材库 + 风格标签匹配
- **推送服务**（二期）
- **成本控制策略**：
  - Plan 4：每日 LLM 预算熔断（`budget:llm:daily`）
  - Plan 9c：内容过滤分级宽容化降低 fallback 率
  - **Plan 11A**：大纲预览-确认模式，被拒大纲只烧小钱（节省整篇故事成本）
  - **Plan 11B**：成本可观测（LLM/TTS/storage 实际成本 metric + `cost_events` 历史表 + `cost-report` CLI），为后期定价提供数据基础
  - 缓存重复元素 / 预生成池 / 按订阅等级限流（**待商业化阶段引入**，Plan 11B 数据成熟后启动）

### 3.6 MEMORY.md —— 长期记忆，跨会话持久化
- **听过的故事**：标题、摘要、关键元素（出场角色、地点、情节钩子）、孩子反馈
- **故事元素索引**：方便彩蛋回调（"上次救的小恐龙"）
- **活跃故事线**：连续剧模式下当前进行中的剧集状态
- **孩子兴趣画像**：基于历史故事的兴趣权重，动态更新
- **家长反馈累计**：哪些主题/风格更受欢迎

### 3.7 HEARTBEAT.md —— 后台定时任务清单
> 一期范围内为轻量版；真·后台预生成与推送在二期实现。

一期任务：
- 每次 App 启动时检查"重逢仪式"是否需要触发
- 每次故事完成后异步更新 MEMORY 与兴趣画像

二期任务（占位）：
- 每天傍晚预生成 1-2 个推送故事候选
- 每周回顾孩子兴趣变化
- 生日/节日特别故事生成

### 3.8 BOOT.md —— 重启时的启动仪式
家长每次打开 App 时执行：
1. 加载 USER.md + MEMORY.md
2. 爱宝在首页主动问候（基于时段、上次互动："小宇晚上好呀～"）
3. 展示"伪推送"卡片（1-2 个爱宝"今天为孩子想的故事"），点开即生成
4. 若上次有未播完故事或活跃连续剧，提示继续

### 3.9 BOOTSTRAP.md —— 首次运行的初始化仪式
**这是最关键的情感锚点。** 第一次打开 App 时执行一次，完成后归档。

流程：
1. 家长完成基础三项（昵称/性别/年龄）
2. 进入"相遇仪式"动画：爱宝从竹林里走出来
3. 爱宝以采访的方式自然问出：
   - 喜欢什么角色/动物？
   - 害怕什么？（明确告知：爱宝答应你，永远不在故事里出现）
   - 家里都有谁？（爸爸妈妈/兄弟姐妹/宠物的名字）
4. 爱宝郑重"约定"：从今天起一起冒险
5. 收集到的信息写入 USER.md（孩子层）

## 4. MVP 功能清单

| 模块 | 内容 |
|---|---|
| 注册档案 | 家长账号 + 1 个孩子档案（昵称/性别/年龄必填） |
| 首次相遇 | BOOTSTRAP 仪式，爱宝采访式收集孩子喜好 |
| 故事生成 | B 模式：一句话输入 + 时长（AI 推断主题/风格 → 大纲卡预览 → 用户确认）—— Plan 11A 取代 Plan 9b 的 C 方案手填旋钮 |
| 故事输出 | TTS 语音朗读 + BGM/音效，无插图 |
| 爱宝出场 | 每个故事里以适配形态陪伴孩子 |
| 记忆 | 记住听过的故事、孩子喜好，生成时复用 |
| 串联 | 默认彩蛋回调；可选"连续剧模式" |
| 重逢 | 再次打开时爱宝主动问候 + 伪推送 |

## 5. 核心用户流程

### 5.1 首次相遇流程（BOOTSTRAP）
家长下载 App → 注册 → 填三项基础信息（昵称/性别/年龄） → 进入相遇动画 → 爱宝采访（孩子可在旁参与，家长代答或共答）→ 约定仪式 → 进入主页

### 5.2 生成故事流程（核心日常流程，Plan 11A 起为 B 模式）

> Plan 11A 之前为"参数旋钮"手填模式，用户反馈意图明显的输入（如"跟奥特曼一起冒险"）AI 应自己听懂。从 Plan 11A 起改为"AI 大纲预览-用户确认"两阶段。

**阶段 1：极简输入**
家长在主页点"生成故事" → 进入生成页：
- **需求输入框**（占主视觉，文字或语音）
- **时长选择**：3 / 5 / 8 分钟（Plan 9c 已下调，默认 5）
- 点 **[让爱宝想想]**

**阶段 2：AI 大纲卡片预览（~2 秒返回）**
后端调 doubao-lite 生成结构化大纲，前端展示卡片：
- 故事标题
- 3-5 句梗概
- 教育意义（1-2 个标签，如"勇气 · 团队合作"）
- 风格（AI 推断，如"冒险探索"）
- 时长（沿用用户选择）

家长三选一：
- **[开始生成]** → 进入正文生成流水线（10-20 秒，老路径不变）
- **[调整]** → 抽屉式微调（改风格/教育意义，不改需求文本）→ 重新生成大纲或直接确认
- **[换个角度]** → 同输入 + nonce 破缓存，重生大纲

**阶段 3：正文生成 + 播放**
拿到 outline_id 后调 `/stories/generate`，进 player 页等待。爱宝思考动画 → TTS+BGM 自动播放 → 故事结束后家长可点赞/换一个/标记不喜欢。

**连续剧模式**：home 页活跃 storyline 卡片入口，跳过大纲（续集承接逻辑见 Plan 8）。

### 5.3 重逢流程（BOOT）
家长再次打开 App → 爱宝问候动画（基于时段+上次记忆）→ 主页展示伪推送卡片 → 家长点开 = 直接进入生成或继续上次

### 5.4 故事串联流程
- **默认彩蛋模式**（始终开启）：每次生成时，AGENTS 中的串联编剧从 MEMORY 抽取近期故事元素，注入 prompt，让 AI 在新故事里自然回调（"还记得上次的小恐龙吗？"）
- **连续剧模式**（家长开启后）：当前故事结尾留钩子；MEMORY 标记"活跃故事线"；下次生成时家长会看到"继续 XXX 的冒险"选项

## 6. 故事生成参数与教育主题库

### 6.1 参数定义

| 参数 | 取值 | Plan 11A 后用户体验 |
|---|---|---|
| **时长** | 3 / 5 / 8 分钟（Plan 9c 下调）| 仍由用户手选——文本无法可靠推断 |
| **风格** | 温馨治愈 / 冒险探索 / 搞笑欢乐 / 神奇魔法 / 科普认知 | **AI 在大纲阶段推断**；用户可在大纲"调整"抽屉手改 |
| **教育主题** | 来自主题库（见 6.2），1-3 个 | **AI 在大纲阶段推断**；用户可改 |
| **梗概 / 教育意义** | 自由文本（AI 生成） | **大纲卡片新增展示**——家长看得见 AI 怎么理解需求 |

### 6.2 教育主题库（50-100 主题）

**六大分类：**

| 大类 | 数量 | 示例 |
|---|---|---|
| 🌱 品格习惯 | ~25 | 勇敢、诚实、分享、坚持、感恩、谦虚、责任… |
| 💝 情绪能力 | ~20 | 接纳害怕、处理愤怒、失败也没关系、想念家人、嫉妒、孤独… |
| 🏠 生活习惯 | ~15 | 独立入睡、刷牙、不挑食、收拾玩具、早睡、洗手… |
| 🤝 社交关系 | ~15 | 交朋友、和兄弟姐妹相处、对长辈有礼貌、说对不起、拒绝… |
| 🌍 认知拓展 | ~15 | 季节、动物、职业、不同国家、简单科学、宇宙… |
| 🎯 成长挑战 | ~10 | 上幼儿园、搬家、迎接弟妹、看医生、第一次… |

**加权与展示逻辑：**
- 按孩子年龄过滤（不显示不适龄主题）
- 按 USER.md "当前挑战"加权置顶
- 按 MEMORY 近期讲过的降权避免重复
- 每次展示 6-8 个，"换一批"按钮换一组
- 支持搜索 + 长按收藏

## 7. 内容安全红线

### 7.1 全球未成年人保护合规基线
- 美国 COPPA（13 岁以下儿童在线隐私保护）
- 欧盟 GDPR-K（儿童数据特殊保护）
- 中国《未成年人网络保护条例》《生成式人工智能服务管理暂行办法》
- 英国 Age-Appropriate Design Code (ICO)
- 苹果 App Store Kids Category 要求
- Google Play Designed for Families 要求

### 7.2 硬约束（写入 AGENTS.md）
**绝对不出现：**
- 真实暴力、血腥、死亡的具象描写
- 恐怖元素（鬼怪、突然惊吓、心理悚然氛围）
- 负面价值观（仇恨、歧视、欺骗他人为乐）
- 任何商业广告、品牌植入
- 政治、宗教、意识形态内容
- 性暗示、不当身体接触
- 自伤、自杀、危险模仿行为
- 真实存在的 IP 角色未授权使用（"奥特曼"等需以"爱宝变身的奥特曼形态"等同人化方式呈现，且仅限对话场景，不商用 IP 资源）
- **该孩子档案中"害怕的东西"清单中的元素**（个性化红线）

### 7.3 软约束
- 冲突可有，但解决方式必须是积极的
- 反派角色不能塑造得"酷"到孩子想模仿
- 失败可以写，但要有出路
- 家长在故事中应被呈现为可信赖、温暖的形象

### 7.4 审核机制
- 生成后由"安全审核 agent"做规则匹配 + LLM 审核双层
- 命中红线则重生成；连续两次命中则降级为预设安全模板

## 8. 技术架构概要

> 本章已通过技术架构 brainstorm 沉淀为独立文档：[2026-04-28-aibao-tech-architecture.md](2026-04-28-aibao-tech-architecture.md)。本节仅作概要。

**关键技术选型：**
- 客户端：Flutter（iOS + Android）
- 后端：Go + Gin + GORM + slog；单体单机部署
- 数据库：PostgreSQL 16；缓存：Redis 7
- 对象存储：腾讯云 COS（**私有 bucket + 签名 URL**）
- LLM：豆包 Pro；TTS：Minimax；短信：腾讯 SMS（均通过 Gateway 抽象层）
- 音频混音：服务端 ffmpeg + 自建 BGM 素材库
- 异步任务：**PG Outbox Pattern**（业务库与事件同事务，确保记忆不丢）
- 服务器：腾讯云香港 2C4G（debug 阶段）

**未决策项：**
- 域名（发布前再决策）
- App Store 上架主体
- 教育主题库 50-100 主题清单
- 真实 IP 法务策略（同人化 + 白/黑名单方案待法务复核）
- 订阅定价
- 大陆正式上线前的境内合规与机房迁移

## 9. 界面设计原则

> 用户特别强调"界面要下功夫"。

### 9.1 整体调性
- **温暖、柔和、像绘本** —— 圆角、低饱和度、手绘感插画、柔光
- **爱宝是视觉锚点** —— 几乎每个页面都有爱宝以不同姿态出现（睡觉、思考、招手、变身）
- **一只手就能用** —— 家长常常一手抱孩子一手操作

### 9.2 主页（生成故事页）设计要点
- 顶部：爱宝大形象 + 一句问候（基于时段和记忆）
- 中部：一个**大且友好**的输入区，文字框 + 大语音按钮（按住说话）
- 中下：参数旋钮（默认折叠成一行可视摘要 "10 分钟 · 温馨治愈 · 勇敢主题"，点击展开）
- 下部：1-2 个伪推送卡片（爱宝今天想的故事）

### 9.3 关键交互细节
- 语音按钮：按住时爱宝做"听"的动作；松开时爱宝做"想"的动作
- 生成等待：爱宝思考动画 + 进度提示词（"爱宝在为小宇编故事呢…""快好啦…"）
- 故事播放页：爱宝以本次故事的形态出现 + BGM 可视化波纹 + 进度条 + 暂停/重播
- 教育主题选择：卡片式横向滑动 + "换一批" 按钮带刷新动画
- 首次相遇仪式：全屏沉浸动画，不要让人觉得是在"填表"

### 9.4 颜色与字体
- 主色：温暖竹叶绿（爱宝胸口能量片色）+ 米白
- 辅色：柔和的睡前蓝、晨光黄
- 字体：圆润、可爱但不幼稚（家长也要看得舒服）
