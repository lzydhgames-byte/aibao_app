## 工程坑列表（按发生顺序）

### Issue 1：Android SDK / NDK 版本下不去

第一次 `flutter run` 报错：
- `flutter_secure_storage` 要 compileSdk 36（项目 35）
- `just_audio` / `audio_session` / `path_provider_android` 要 NDK 27.0.12077973

**修复**：`app/android/app/build.gradle.kts` 写死 `compileSdk = 36` + `ndkVersion = "27.0.12077973"`。Commit: `5420b3f`。

### Issue 2：minSdkVersion 21 vs 23

manifest merger 报错：`flutter_secure_storage` 要 minSdk ≥ 23。
**修复**：写死 `minSdk = 23`（Android 6.0，覆盖 99%+ 设备）。同 commit。

### Issue 3：AndroidManifest 缺 INTERNET 权限

APK 装上了但 dio 调不出去，错误 `errno=1 EPERM`。
**根因**：Flutter 默认 manifest 在 **release 模式不自动注入** `<uses-permission android:name="android.permission.INTERNET"/>`。debug 模式 Flutter tool 会隐式注入。
**附带**：Android 9+ 默认禁明文 http，需要 `android:usesCleartextTraffic="true"`（prod 上线必须改 https 并删此项）。
**修复**：同 commit `5420b3f`。

### Issue 4：adb reverse + baseUrl 不匹配

初始 baseUrl 是 `http://10.0.2.2:8080`（**仅模拟器** 才识别的特殊 host）。真机上手机的 localhost 是手机自己。
**修复**：改成 `http://127.0.0.1:8080` + 加 `adb -s emulator-5558 reverse tcp:8080 tcp:8080` 反向隧道。Commit: `863ec05`。

### Issue 5：登录屏 layout bug

首次进入时"发送验证码"按钮**看不见**。原因：`Row(children: [Expanded(TextField), SizedBox(button)])` + `crossAxisAlignment: stretch` 在 OPPO ColorOS Material 3 下渲染出宽度 0 的 input。
**修复**：拆成纵向两 row（输入框一行、按钮一行）。Commit: `30015ae`。

### Issue 6：ExoPlayer release-mode NPE（最难的一个）

网络全通后，just_audio 的 `setUrl()` 抛 `java.lang.NullPointerException`，stack 指向 `r1.a.g (Unknown Source:1)`——R8 混淆后的类名。

logcat 关键行：

```
ExoPlayer:Loade: type=1400 audit(0.0:13463): avc: denied { connectto }
for path="/dev/socket/dnsproxyd"
scontext=u:r:untrusted_app:s0:...  tcontext=u:r:init:s0
tclass=unix_stream_socket  permissive=1  app=com.aibao.aibao_app
```

**真相**：ExoPlayer 的 "Loader" 线程（logcat 截断为 `ExoPlayer:Loade`）在 release 模式被 R8 重命名 → OPPO ColorOS 自定义 SELinux 策略**没把这个新线程名加白名单** → DNS query 通过 `dnsproxyd` 被 denied → 返回 null → ExoPlayer NPE。

**验证**：PC curl + Android shell `curl` 直接下同一个 COS 签名 URL 都成功——系统 DNS / TLS / 网络全部 OK。**只在 ExoPlayer R8 混淆线程里翻车**。

**绕过办法**：`flutter run`（debug 模式，不混淆）。Play Store / 应用市场签名 release apk 的 obfuscation seed 不同，且多数 AOSP 设备走标准 SELinux 策略，**生产用户不会遇到**。极端情况可用 `proguard-rules.pro` keep ExoPlayer 内部类。

不需 code commit；只是 `flutter run` 去掉 `--release` flag。

### Issue 7：声音选型（A/B/C 三轮）

配置文件改 `voice_id`，server restart，不需要客户端 rebuild：

- `female-tianmei`（最初默认）— 用户："声音不太好"

- `audiobook_female_2`（温暖成熟）— 备选

Commit: `667f3b4`。

## 工程教训沉淀（写到知识库 + MEMORY.md）

### 1. 前端 + 后端首次联调的 6 类 Android 工程坑

每一类都有明确解法但首次踩需要小时级 debug：
- SDK 版本不匹配 plugin（compileSdk 36）
- NDK 版本不匹配 plugin（27.0.12077973）
- AndroidManifest 缺 INTERNET 权限（release 不自动注入）
- cleartextTraffic（http 必须显式开）
- adb reverse 与 emulator-only `10.0.2.2` 的区别
- 国产 ROM SELinux + R8 混淆联动 NPE

详见知识库新建 `12-flutter.md`。

### 2. flutter run --debug vs --release 取舍

调试用 debug，**release 留到 Plan 10 部署前再彻底验证**。很多 plugin 行为只在 debug 模式被 Flutter tool 修补；release 模式必须显式声明所有东西。直觉用 `--release` "更接近生产" 反而坑多。已记录知识库 12.7。
