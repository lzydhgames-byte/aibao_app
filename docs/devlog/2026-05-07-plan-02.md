### 关键技术决策落地

| 决策 | 实现 |
|---|---|
| SMS 一期仅 Mock | sms.NewMock() FixedCode=123456 + dev 日志 |
| 注册流程：两步 + 自动注册 | 第一次手机号 → CreateOrGet 新建；之后登录 |
| 一期单孩子 | DB UNIQUE(user_id) + repo 转 ErrAlreadyExists → service 转 409 |
| JWT | HS256 + access 24h + refresh 7d + Type 字段防混用 |
| 手机号双存 | safehash(phone) 用于查询 + AES-256-GCM 加密原文 |
| 错误码解耦 | service 返 AppError → api.RespondError 自动映射 HTTP 状态 |

## 遇到的问题与解决

1. **testcontainers v0.42 API 变化** —— `WithWaitStrategyAndDeadline` 弃用，改 `WithWaitStrategy(wait.ForLog/ForListeningPort)`（Plan 1 末尾就遇到过）
2. **golangci-lint flagged `autoMigrateForTest` as unused** —— 它只被集成测试用，拆到独立 `migrate_testhelper.go` + `//go:build integration` tag
3. **Smoke 测试 404** —— `bin/aibao-server` 是 Plan 1 时编译的旧二进制，Task 17 修改 router 后没重新 build。`go build` 后立即恢复
4. **Smoke 测试 400 invalid_argument** —— Windows 全局 HTTP 代理 (`http_proxy=127.0.0.1:18081`) 拦截了 curl 请求并改了 body。加 `--noproxy "*"` 绕过
