### ✅ 完成：技术架构 brainstorming，输出技术架构文档

延续产品 spec，与产品负责人完成技术架构对齐。

**关键产出：**
- [技术架构设计文档](../superpowers/specs/2026-04-28-aibao-tech-architecture.md)

**技术栈定型：**
- 客户端 Flutter / 后端 Go+Gin+GORM / DB PG 16 / 缓存 Redis 7 / 存储腾讯 COS
- LLM 豆包 Pro / TTS Minimax / 短信腾讯 SMS（均通过 Gateway 抽象层接入）
- 服务器：腾讯云香港 2C4G/70GB/30Mbps，承载 ~1000 测试用户
- 部署：systemd + Nginx 单机单体，不上 Docker（YAGNI）

**强制贯穿原则（写入文档）：**
- 三层架构（api/service/repository）+ Gateway 抽象外部依赖
- 配置外置、文件直走 COS、schema 走 migrate、备份脚本入 git
- 日志 slog JSON + traceId 贯穿全链路 + 敏感字段脱敏
- 为未来迁移而设计——服务器当"牲口"不当"宠物"

**特别讨论的产品负责人成长向内容：**
应产品负责人要求，技术架构文档中嵌入了 🎓 软件工程概念讲解（YAGNI / 三层架构 / 依赖倒置 / 幂等 / 优雅关停 / Pets vs Cattle 等），便于学习。

### ✅ 完成：基于 Codex review 的技术架构 v1.1 重构

产品负责人用 Codex 对 v1.0 做了一轮独立 review，提出 P1×3、P2×5 共 8 项问题。全部接受并落地。

**主要变更：**

1. **架构第 0 章新增"必须守住的四条线"**：孩子是主角 / 爱宝人格一致 / 儿童数据默认敏感 / 安全链路可验证（来自 Codex 总结）
2. **异步任务改 PG Outbox Pattern**（P1-1）：业务库与事件同事务；Redis 仅做唤醒通知；含 DLQ + replay；解决"BRPOP 后崩溃丢消息"
3. **音频私有化**（P1-2）：COS 私有 bucket；存 `audio_object_key` 不存裸 URL；新增 `Storage.SignedURL` 接口，播放走 `GET /api/v1/stories/{id}/audio_url` 鉴权后签发 15 分钟链接
4. **安全链路双层化**（P1-3）：前置 PreCheck（规则 + 害怕清单 + 真实 IP 归一化 + 意图分类）+ 强约束 System Prompt 模板（版本化）+ 后置 PostCheck（含主角身份校验）
5. **新增第 8 章音频编排**（P2-1）：LLM 输出标记 → cue_parser → TTS → ffmpeg 混音；多级降级；BGM 素材库表 `bgm_assets` 含 license 字段
6. **新增第 11 章测试策略**（P2-2）：MVP 必测清单（prompt_builder / 安全规则 / 重生成降级 / 记忆幂等 / 限流 / 预算熔断 / 签名 URL / 日志脱敏 / 导出删除）；service 层 ≥70%；端到端冒烟脚本
7. **新增第 10.3 节 Metrics**（P2-3）：Prometheus 客户端暴露 `/metrics`（仅 127.0.0.1），覆盖 story_generate / llm_call / tts_call / safety_fail / outbox_pending / budget_used 等 SLO 指标；Prometheus server 暂不部署
8. **children 表加 UNIQUE(user_id)**（P2-4）：一期单孩子 DB 层保护；二期开放路径已写入演进路线
9. **新增附录 C：12 条 ADR**（P2-5）：客户端/后端/LLM/TTS/COS/异步/部署位置/单体/Docker/Metrics/双层安全/单孩子约束 各有候选与回滚条件
10. **同步更新产品 spec 第 8 章**：消除"待决策"与已定案的矛盾，指向技术架构文档
