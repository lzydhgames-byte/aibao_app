## Issue 与教训（2 个 latent bug）

### Bug 1：outbox 事件 payload 后改不生效（commit 4b522f3）

- **现象**：Plan 6 加 `memories.story_id` FK 后，memory_update Worker 写 memory 时 FK 违反——payload 里 `story_id` 永远是 0。
- **根因（Plan 4 埋的）**：Orchestrator 在事务里先 INSERT outbox event（此时 story.ID 还没生成）→ CreateWithOutbox 返回后才"patch Go 内存里的 payload.story_id = story.ID"。但 outbox row 已经写出去了，patch 只改了内存，**DB 永远是 `story_id: 0`**。
- **Plan 4 为什么没爆**：那时 memory_update handler 只读 payload 的 title / summary，没读 story_id——bug 静默。
- **Plan 5 怎么躲的**：tts_synthesis handler 早就用 `e.AggregateID` 兜底（CreateWithOutbox 内部正确填了 aggregate_id）。
- **Plan 6 怎么暴露的**：新 memory FK 需要 story_id → handler 第一次真读 payload → 取到 0 → FK 违反。
- **修复**：memory_update handler 改成 "优先 `e.AggregateID`，payload.story_id 仅做兜底"，复用 Plan 5 已验证的模式。
- **沉淀知识库 5.16**。

### Bug 2：intent_model endpoint 配错了 LLM（commit a58a774）

- **现象**：Bootstrap 润色 + Memory summary（都跑 intent_model）调用 LLM 全部失败，但应用 fail-open 让用户感知不到。
- **根因（Plan 4 埋的）**：当时为省钱把 `intent_model` 绑到 Doubao-Seedream-5.0-lite——但 **Seedream 是文生图模型，不接受 chat completion**。意图分类调用全 404，fail-open 默认 IntentSafe → 功能"看似正常"。
- **Plan 4/5 为什么没爆**：意图分类挂了走 fail-open 到 safe，**业务逻辑无感**。
- **Plan 6 怎么暴露的**：Bootstrap polish + Memory summary 让 intent_model 第一次真正干活——LLM 失败=数据缺失=用户可见。
- **修复**：火山引擎控制台新建 endpoint `ep-20260511153638-cstj9` 绑 doubao-1.5-lite-32k → 替换 config。
- **沉淀知识库 9.14**——fail-open 必须配指标告警，否则 latent bug 永远不显形。

## Known Issues（留给 Plan 6b / 7）

1. **BOOTSTRAP 润色不传 nickname** → LLM 编造昵称"小明"。改 `renderDescription` 传入 child.Nickname。
2. **PostCheck `minProtagonistOccurrences=3` 太严** → 5 分钟短故事里 nickname 出现 1-2 次很常见，被打成 `child_not_protagonist`。考虑按 duration 缩放（5min=2 / 10min=3 / 15min=4）。
3. **软提示彩蛋不工作** → Selector 注入了 675 字 memory context，LLM 仍然忽略。改进路径：
   - 把 memory 段从 system prompt 末尾抬到 `## 上下文` 单独章节
   - 措辞从"可以自然回调"改为"请尝试在新故事中借用之前出现过的角色或场景"
   - 极端方案：加一轮"编辑 LLM call"检查回调出现，否则 force regenerate
   - 沉淀知识库 11.10
4. **child nickname 编码遗留**（Plan 4 Git Bash GBK bug）：id=3 的 nickname 是乱码 "С��"，手动 `UPDATE children SET nickname='小宇'` 修复。后续 PowerShell + UTF-8 创建的孩子无问题。
