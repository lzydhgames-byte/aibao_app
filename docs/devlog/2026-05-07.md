### Plan 2 端到端 smoke 测试通过

```
1) POST /api/v1/auth/sms/send {phone:13400134000}
   → 200 {"sent":true}
   服务端日志：sms.mock.send code:123456 phone:138****4000

2) POST /api/v1/auth/login_or_register {phone, code:123456, nickname:小宇妈妈}
   → 200 {access_token, refresh_token, user:{id:6, nickname:"小宇妈妈"}}

3) GET /api/v1/me  Authorization: Bearer <access>
   → 200 {id:6, nickname:"小宇妈妈"}

4) POST /api/v1/children {nickname:小宇, gender:boy, birthday:2020-08-15}
   → 201 {id:1, user_id:6, nickname:"小宇", ...}

5) POST /api/v1/children 第二次
   → 409 {reason:"child_already_exists"}（DB UNIQUE 起作用）

6) GET /api/v1/children
   → 200 {items:[{...}]} 长度 1

7) PATCH /api/v1/children/1 {nickname:小宇宙}
   → 200 {nickname:"小宇宙", ...}

8) GET /api/v1/me （不带 token）
   → 401 {reason:"unauthorized"}
```

### Plan 2 关键技术决策落地

| 决策 | 实现 |
|---|---|
| SMS 一期仅 Mock | sms.NewMock() FixedCode=123456 + dev 日志 |
| 注册流程：两步 + 自动注册 | 第一次手机号 → CreateOrGet 新建；之后登录 |
| 一期单孩子 | DB UNIQUE(user_id) + repo 转 ErrAlreadyExists → service 转 409 |
| JWT | HS256 + access 24h + refresh 7d + Type 字段防混用 |
| 手机号双存 | safehash(phone) 用于查询 + AES-256-GCM 加密原文 |
| 错误码解耦 | service 返 AppError → api.RespondError 自动映射 HTTP 状态 |
