## Issue 与教训

### Issue 1：Git Bash + curl 中文编码污染
- **现象**：`curl -d '{"prompt":"血腥..."}' http://localhost:8080/...` 期望被红线拦截，实际穿透到 LLM 调用层。
- **根因**：Git Bash on Windows 默认 locale 是 GBK / CP936。bash 把命令行参数中的 UTF-8 字符串当作 GBK 字节交给 curl；服务端按 UTF-8 解码 → 乱码字节，红线词匹配不到。
- **影响**：仅影响 **本地 smoke 测试**。生产 Flutter / iOS 客户端的 HTTP 库永远 UTF-8 序列化 JSON，不受影响。
- **绕法**：用 PowerShell + `Invoke-WebRequest` + `[System.Text.Encoding]::UTF8.GetBytes($body)` 显式发 UTF-8。
- **NEXT**：写一个 `scripts/smoke.ps1` 固化 PowerShell 调用方式，避免下次再踩。

### Issue 2：火山引擎 Endpoint ID 系统
- **现象**：第一次接入豆包 OpenAI 兼容 API 时，`model: "doubao-1.5-pro-32k"` 返回 404。
- **根因**：Volcengine 不接受模型名称——必须先在控制台创建"推理接入点"绑某个具体模型，得到 `ep-m-XXXXX` 形式的 endpoint ID，**该 ID 才是 API 调用时填的 model 字段**。
- **影响**：每次接入新模型都要走"创建 endpoint → 拷 ep-m- ID → 写进配置"流程。
- **NEXT**：config.dev.yaml 的 `story_model` 和 `intent_model` 都已更新为 ep-m- 格式；新接入模型时记得走同样流程。

### Issue 3：意图分类 endpoint 选错
- **现象**：每次生成请求 LLMIntentProvider 调用都 404。
- **根因**：用户初次创建的 intent endpoint 绑了 **Seedream-5.0-lite（一个文生图模型）**，对纯文本分类调用返回 404。
- **影响**：LLMIntentProvider 失败 → fail-open 到 IntentSafe → 无功能影响，但每次请求多浪费一次 LLM 调用尝试。
- **NEXT**：在控制台创建一个真正的纯文本 lite endpoint（如绑 doubao-1.5-lite-32k），更新 config.dev.yaml 的 `intent_model`。

### Issue 4：业务 metrics 注册但未埋点
- **现象**：`/metrics` 上 9 个业务 metric 都在，但 counter 永远 0。
- **根因**：main.go 中 `bm := metrics.NewBusiness(reg)` 构造完后写了 `_ = bm`——没有注入 Orchestrator 和 middleware。
- **影响**：Plan 4 主目标"端到端故事生成"已达成，监控数据空一段时间可接受。
- **NEXT**：Plan 5/6 把 `bm` 注入 Orchestrator + 限流/预算 middleware，让计数真的发生。
