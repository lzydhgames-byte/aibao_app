### Plan 9b 收尾（早上 → 下午）

Plan 9b（Flutter UI 完成）所有 Task 0–13 都在前一天写完，今天是真机 smoke + 一连串 Bug 修复的拉锯。

#### 修了 4 个 Riverpod / dio 状态管理坑（uncommitted → 全部 commit）

1. **AsyncNotifier 缓存幽灵**（commit `bc13d2c`）
   - 现象：换手机号登录后还看到上一个用户的孩子档案
   - 根因：`childProvider` 是 `AsyncNotifier`，session 切换不会自动 invalidate
   - 修法：把 `Ref` 注入 `AuthNotifier`，在 login / logout 显式 `ref.invalidate(childProvider)`

2. **401 反馈环爆炸**（commit `cfa1108`）
   - 现象：未登录冷启动后日志一秒打几百条 `GET /children` 401，前端 toast"请稍后再试"
   - 根因：`childProvider.build()` → 401 → logout → invalidate → 重 build → 401 → 死循环
   - 修法：401 interceptor 在调用 logout 前检查 `auth is AuthAuthenticated`，避免对已经登出的 session 重复回调

3. **CircularDependencyError**（commit `32a3c71`）
   - 现象：runtime 抛 Riverpod 循环依赖
   - 根因：`apiClientProvider.onUnauthorized` 里 `ref.read(authProvider)` → `AuthNotifier` 构造时又依赖 `apiClient`
   - 修法：引入 `authSnapshotProvider` —— 一个**无依赖**的可变 Bool 中间层。`AuthNotifier` 通过覆盖 `set state` 写入快照；401 interceptor 同步读快照，从不触达 `authProvider`

4. **SMS 验证码"一次性消费"反人类**（commits `2488d85`）
   - 现象：输错一次验证码后，输对的也提示"已过期"
   - 根因：`codeStore.Take()` 用 Redis `GETDEL` 原子操作，错的也消费掉
   - 修法：拆成 `Peek`（GET 不删）+ `Consume`（DEL，只在匹配成功后调）

#### 路由作用域 bug：限流误伤读接口（commit `6b545fd`）

- 现象：生成故事后立刻进 player 屏，几次刷新就触发 429
- 根因：`auth.Group("")` 子组挂了 5/min 限流，但 Story handler 把 **所有路由**（包括 `GET /stories/:id`）注册到这个子组上
- 修法：`RegisterRoutes` 接收 variadic `genGuards`，**只**给 `POST /stories/generate` 这一条挂限流

#### 列表缓存失效漏写（commit `1c41cf6`）

- 现象：从生成页跳到 player 后回到首页，"最近听过"里没有新故事
- 根因：`storyListProvider` 是 `FutureProvider.family`，没有任何位置 invalidate
- 修法：跳转 player 前 `ref.invalidate(storyListProvider(child.id))`

### Plan 9c 故事质量启动（傍晚 → 下班前）

#### Task 1：时长校准 + 长度保护双管齐下

**起因**：用户反馈选 5 分钟挡音频只有 2:06。

**诊断**：`expectedRunesForDuration` 系数是 120 字/分钟，但 Minimax t2a_v2 实测速率 ~300 字/分钟。LLM 即使按 600 字写满也只够 2 分钟。

**修法链**：
1. 系数 120 → **320 字/分钟**（commit `bf084ee`）
2. prompt 「约 X 字」→ **硬约束 ±10% 范围**（commit `bf084ee`）
3. 字数翻 2.5 倍后 LLM 调用 60s 超时被打爆 → 后端 timeout 30 → 90s（commit `ec3af74`），客户端 dio 60s → 120s（commit `2fa6bc4`）
4. LLM 仍偶尔欠写（1118 字目标 2304） → 加 **长度保护 + 1 次重写**（commit `7d84c81`）
5. 实测仍欠 → 重试上限 1 → 2，dio 120 → 180s（commit `62b8613`）

#### Task 2：挡位调整 5/10/15 → 3/5/8（commit `aa99d6a`）

用户反馈 10/15 分钟太长，改为 3/5/8。涉及 6 层：API validator、prompt 测试、fallback 文件重命名、Flutter chip 选项、CLI 默认值、5 个测试文件 mock 数据。

#### Task 3：同质化破除（commits `ec33a63` + `a6e2cbf`）

**现象**：所有故事开头都是「夜晚，月光洒在房间里，爱宝蹦蹦跳跳来到小宇床边」。

**双管齐下**：
1. **SceneSeed 池子**：20 → 80 个场景种子，9 大类（时间片段 / 天气 / 日常场所 / 玩乐场所 / 季节节日 / 户外自然 / 交通工具 / 家里亲人 / 想象奇遇 / 学习兴趣）。`builder.Build()` 每次随机抽一个注入 prompt。
2. **反套路条目**：prompt 明文写「**不要**每个故事都以"夜晚/床边"开场」+「**不要**用"今天咱们去 XX 吧"句式」。

效果：标题立刻出现"端午节 / 星星城堡 / 草原星空"等多样化。

#### Task 4：续集 PostCheck warn-only（commit `96a86f4`）

**现象**：3 分钟续集音频只有 45 秒。

**诊断**：Plan 8 加的"续集必须命中至少 1 个上一集元素"硬约束触发，故事走 fallback。3min fallback 模板只有 ~150 字 → 朗读 45 秒。

**修法**：续集连续性 reject 从 hard-fail 改为 **warn-only**。其他 PostCheck 失败（红线 / 主角错位 / 恐惧词）仍 hard-fail。

#### Task 5：红线词「刺」误伤（commit `5549e87`）

**现象**：8 分钟故事第一次走通长度重写到 1879 字，PostCheck 命中「刺」红线，掉 fallback。

**诊断**：单字「刺」过宽，会误伤"刺猬 / 玫瑰花的刺 / 刺啦一声 / 刺绣"等无害童话词。

**修法**：删除单字「刺」，保留意图明确的「刺死 / 捅 / 捅死」。

## 今日学到的核心教训

1. **每改一层都要回溯整个调用链的超时栈**：今天为了字数翻倍连改 dio / orchestrator / LLM gateway / dio 共 4 次超时。
2. **硬约束在产品演进里可能反客为主**：Plan 8 的续集连续性硬约束 + Plan 9c 的字数硬约束叠加 → 用户拿到 45 秒罐头模板。每条硬约束都要问"失败时降级到的东西是不是更糟"。
3. **safety filter 不能用单字**：单字模式无视上下文，红线词必须用最小完整意图组合。
4. **"假失败" `flutter run`**：PowerShell 后台模式下 `flutter run` 总在 VM Service 握手处报"failed"，但 APK 已经装到手机上。看到这个错误不代表打包失败，看 `√ Built ... apk` + `Installing ... apk` 即可。
